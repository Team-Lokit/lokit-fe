## 1. 컴포넌트 렌더링 검증

> **검증 방법**: `pnpm storybook` 실행 후 각 컴포넌트 스토리에서 확인
>
> - `Components > Sidebar` — Sidebar 전체 (Default, EmptyAlbums, ManyAlbums)
> - `Components > Sidebar > AlbumListItem` — 개별 아이템 (Default, Selected, WithMenu, LongTitle)
> - `Components > ViewSwitcher` — 뷰 전환 토글 (MapView, GridView)

### 1.1 Sidebar 구조

- [ ] Sidebar가 화면 좌측에 오버레이로 표시되는가
- [ ] 고정 너비 270px이 적용되는가
- [ ] 높이가 화면 전체(100vh)를 채우는가
- [ ] 배경색 `gray.a80` (`rgba(36,36,38,0.8)`)이 적용되는가
- [ ] `backdrop-filter: blur(40px)`이 적용되는가
- [ ] `border-right: 1px solid blueWhite.border10` (`rgba(226,230,255,0.10)`)이 적용되는가
- [ ] z-index가 `zIndex.overlay` (1100)로 설정되어 모달(1200)보다 낮은가

### 1.2 Header

- [ ] "LOKIT" 워드마크 SVG(`lokitWordmark.svg`)가 좌측에 `primary.400` (`#6EEAE4`) 색상으로 표시되는가
- [ ] 워드마크 크기가 60×14px인가
- [ ] ✕ 닫기 버튼(`close.svg`, 20×20px)이 우측에 표시되는가
- [ ] 패딩이 `pl: 12px, py: 8px`인가

### 1.3 SearchInput

- [ ] "앨범을 검색해보세요..." placeholder가 `gray[400]` 색상으로 표시되는가
- [ ] 검색 아이콘(`search.svg`, 16×16px)이 표시되는가
- [ ] 배경색이 `blueWhite.bg5` (`rgba(226,230,255,0.05)`)인가
- [ ] `border-radius: 12px`, 패딩 `px: 12px, py: 10px`이 적용되는가
- [ ] 기존 `Input` 컴포넌트가 재사용되었는가
- [ ] 입력 필드가 정상적으로 포커스/입력 가능한가

### 1.4 Menu

- [ ] "탐색" 메뉴가 `explore.svg` (20×20px) 아이콘과 함께 표시되는가
- [ ] "새 앨범" 메뉴가 `newAlbum.svg` (20×20px) 아이콘과 함께 표시되는가
- [ ] 메뉴 아이템 간 gap이 `4px`인가
- [ ] 아이템 텍스트가 `body-16-medium`, `text.primary` (`#D3D3D3`)인가

### 1.5 AlbumList

- [ ] "앨범" 라벨이 `body-14-regular`, `gray[500]` (`#7B7B7D`)로 표시되는가
- [ ] 전체사진 앨범이 최상단에 `blueWhite.bg8` 배경 강조 스타일로 표시되는가
- [ ] 전체사진 앨범에 사진 개수가 표시되는가
- [ ] 사용자 생성 앨범에 제목 + 사진 개수가 표시되는가
- [ ] 사용자 생성 앨범에 더보기(···) 메뉴가 표시되는가
- [ ] 전체사진(기본 앨범)에는 더보기 메뉴가 없는가
- [ ] AlbumListItem 패딩이 `px: 12px, py: 10px`, gap `20px`, `border-radius: 12px`인가
- [ ] hover/pressed 시 `blueWhite.bg8` 배경 디졸브 효과가 적용되는가

### 1.6 My

- [ ] 하단에 고정 배치되는가 (`margin-top: auto` 또는 flex 레이아웃)
- [ ] 상단 구분선 `border-top: 1px solid blueWhite.border10`이 있는가
- [ ] 프로필 원형이 `44px` 크기이며 `blueWhite.bg8` 배경 + `backdropBlur[25]`가 적용되는가
- [ ] 프로필 이미지(키 아이콘 아바타)가 `36px` 원형으로 표시되는가
- [ ] 닉네임이 `body-16-medium`, `gray[100]` (`#E8E8E8`)로 표시되는가
- [ ] D-day 뱃지가 `blueWhite.bg8` 배경, `border-radius: 8px`인가
- [ ] D-day 텍스트가 `body-15-medium`, `gray[300]` (`#B2B2B4`)이며 하트 아이콘(`12px`)과 함께 표시되는가

## 2. 인터랙션 검증

### 2.1 사이드바 열기/닫기

- [ ] ≡ 햄버거 버튼 클릭 시 사이드바가 열리는가
- [ ] ✕ 버튼 클릭 시 사이드바가 닫히는가
- [ ] 사이드바 외부(backdrop) 클릭 시 닫히는가
- [ ] ESC 키 입력 시 닫히는가

### 2.2 열기/닫기 애니메이션

- [ ] 열릴 때 좌측에서 슬라이드 인 (`translateX(-100%) → translateX(0)`)이 동작하는가
- [ ] 닫힐 때 좌측으로 슬라이드 아웃이 동작하는가
- [ ] Backdrop이 fade in/out 되는가
- [ ] transition이 `0.3s ease`로 부드럽게 동작하는가

### 2.3 검색 기능

- [ ] 검색어 입력 시 앨범 목록이 실시간 필터링 되는가
- [ ] 필터링 로직이 앨범 제목 포함 여부 기반인가
- [ ] 검색어 삭제 시 전체 앨범 목록이 복원되는가
- [ ] 사이드바 닫았다 열면 검색어가 초기화되는가
- [ ] 한글 검색이 정상 동작하는가 (조합 중 필터링 이슈 없음)

### 2.4 메뉴 동작

- [ ] "탐색" 클릭 시 탐색 페이지로 이동하는가
- [ ] "새 앨범" 클릭 시 앨범 추가 모달이 열리는가

### 2.5 앨범 리스트 동작

- [ ] 전체사진 앨범 클릭 시 해당 앨범 상세(격자뷰)로 이동하는가
- [ ] 사용자 앨범 클릭 시 해당 앨범 상세(`/album/:id`)로 이동하는가
- [ ] 더보기(···) 메뉴 클릭 시 드롭다운(이름 변경/삭제)이 표시되는가
- [ ] "앨범 이름 변경" 클릭 시 이름 변경 모달이 열리는가
- [ ] "앨범 삭제" 클릭 시 삭제 확인 모달이 열리는가

### 2.6 My 영역 동작

- [ ] My 영역 클릭 시 마이페이지로 이동하는가

## 3. 상태 관리 검증

### 3.1 사이드바 상태

- [ ] 사이드바 열림/닫힘 상태가 홈 화면 레벨에서 관리되는가
- [ ] 사이드바 열림 시 지도/격자뷰가 뒤에 유지되는가
- [ ] 사이드바 열림 시 배경 스크롤이 잠기는가
- [ ] 라우트 변경 시 사이드바가 적절히 닫히는가

### 3.2 앨범 데이터

- [ ] 앨범 추가 후 사이드바의 앨범 목록에 즉시 반영되는가
- [ ] 앨범 이름 변경 후 목록에 즉시 반영되는가
- [ ] 앨범 삭제 후 목록에서 즉시 제거되는가
- [ ] Pending 사진(업로드 중)이 앨범 카운트에 반영되는가

## 4. 뷰 전환 검증

### 4.0 ViewSwitcher

- [ ] pill 형태의 토글이 표시되는가
- [ ] 활성 탭에 아이콘 + 텍스트가 표시되고, 비활성 탭에 아이콘만 표시되는가
- [ ] 활성 탭 배경이 `gray[100]` (`#E8E8E8`)인가
- [ ] 배경에 `blueWhite.bg8` + `backdropBlur[25]` + `blueWhite.border10` 테두리가 적용되는가
- [ ] 지도보기 클릭 시 지도뷰로 전환되는가
- [ ] 격자보기 클릭 시 격자뷰로 전환되는가
- [ ] 지도보기 아이콘이 `mapPin.svg`, 격자보기 아이콘이 `grid.svg`인가

### 4.1 지도뷰

- [ ] 사이드바가 닫힌 상태에서 지도가 전체 화면으로 표시되는가
- [ ] 지도 위 사진 핀이 정상 표시되는가
- [ ] 하단 "기록 N개" 플로팅 버튼이 표시되는가
- [ ] "격자보기" 버튼으로 격자뷰 전환이 되는가
- [ ] 사이드바를 열 수 있는 트리거(≡ 버튼)가 존재하는가

### 4.2 격자뷰

- [ ] 상단 헤더에 ≡ 버튼, "전체사진" 제목, 🔔 알림 버튼이 표시되는가
- [ ] "기록 N개" 뱃지가 표시되는가
- [ ] 사진이 없을 때 빈 상태 UI가 표시되는가 ("기록이 없어요")
- [ ] 빈 상태에서 "사진 추가", "앨범 추가" 카드가 동작하는가
- [ ] 하단에 ViewSwitcher(지도보기/격자보기 토글)가 표시되는가

## 5. 기존 기능 회귀 검증

> BottomSheet 제거 후 기존 기능이 정상 동작하는지 확인

### 5.1 사진 관련

- [ ] 사진 추가 플로우가 정상 동작하는가 (파일 선택 → 노트 작성 화면 이동)
- [ ] 지도 핀 클릭 → 사진 상세 이동이 정상인가
- [ ] 클러스터 핀 클릭 → 클러스터 상세 이동이 정상인가

### 5.2 앨범 관련

- [ ] 앨범 추가 모달이 정상 동작하는가
- [ ] 앨범 이름 변경 모달이 정상 동작하는가
- [ ] 앨범 삭제 모달이 정상 동작하는가
- [ ] 앨범 상세 페이지(`/album/:id`) 진입/이탈이 정상인가

### 5.3 지도 관련

- [ ] 현재 위치로 이동 기능이 동작하는가
- [ ] 지도 뷰 상태(줌, 위치)가 유지되는가
- [ ] 위치 권한 거부 시 안내 모달이 정상 표시되는가

### 5.4 헤더 관련

- [ ] 주소 표시가 정상인가
- [ ] 프로필 이미지 표시가 정상인가
- [ ] 앨범 상세 시 뒤로가기/메뉴 헤더가 정상인가

---

## 6. 엣지 케이스 검증

- [ ] 앨범이 0개일 때 사이드바가 정상 표시되는가 (전체사진만 표시)
- [ ] 앨범이 많을 때(20개+) AlbumList 영역 스크롤이 정상 동작하는가
- [ ] 앨범 이름이 매우 길 때 말줄임(ellipsis) 처리되는가
- [ ] 사진 개수가 매우 클 때(10000+) 숫자 표시가 깨지지 않는가
- [ ] 빠르게 열기/닫기 반복 시 애니메이션이 깨지지 않는가
- [ ] 사이드바 열린 상태에서 브라우저 뒤로가기 시 적절히 처리되는가
- [ ] Pending 사진(업로드 중)이 앨범 카운트에 정상 반영되는가

---

## 7. 성능 검증

- [ ] 사이드바 열기/닫기 시 지도 컴포넌트의 불필요한 재렌더링이 발생하지 않는가
- [ ] 검색 입력 시 불필요한 재렌더링이 없는가 (메모이제이션 활용 확인)
- [ ] 앨범 목록이 많을 때 렌더링 성능이 적절한가

---

## 8. 접근성 검증

- [ ] `<aside>` 시맨틱 태그가 사용되었는가
- [ ] 닫기 버튼에 `aria-label`이 있는가
- [ ] 키보드 탐색(Tab)이 정상 동작하는가
- [ ] 스크린리더가 사이드바 컨텐츠를 올바르게 읽는가

---

## 9. 테마 토큰 검증

- [ ] `blueWhite.border10` 값이 피그마 기준 `rgba(226,230,255,0.10)`으로 수정되었는가
- [ ] 기존 `blueWhite.border10` 사용처에 사이드 이펙트가 없는가 (BottomSheet 제거 후 무관)
- [ ] Sidebar에 사용된 모든 색상이 `theme.colors`에서 참조되는가 (하드코딩 없음)
- [ ] Sidebar에 사용된 효과가 `theme.effects`에서 참조되는가
- [ ] Sidebar 너비가 `theme.layout.sidebarWidth`를 참조하는가 (하드코딩 없음)
